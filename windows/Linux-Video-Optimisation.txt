[To remove NVIDIA drivers from the Ubuntu 'initrd' (initial RAM disk) image]
----------------------------------------------------------------------------
 - you must perform two main steps: [1] uninstalling the driver packages from the system and [2] regenerating the initrd file to reflect those changes. 

[1] Uninstall NVIDIA Drivers 
First, remove the actual driver files and configurations from your system so they are not available to be included in the next image build. 
 - For drivers installed via 'apt' (standard):
   Run the following command to purge all NVIDIA-related packages and dependencies:
      :~$ sudo apt-get purge '^nvidia-.*'
    or
      :~$ sudo apt remove --purge nvidia-*
   Follow this with:
      :~$ sudo apt-get autoremove
   If get errors about unmet dependencies, try to fix them with:
      :~$ sudo apt --fix-broken install
 - For drivers installed via '.run' file (manual):
   If you used a manual installer, run the uninstall script:
      :~$ sudo nvidia-uninstall
   (Note: If the file is still on your system, you can also run "sudo ./NVIDIA-Linux-x86_64-<version>.run --uninstall")

[2] Update the initrd Image 
The initrd image is a static archive created at a specific point in time. Simply deleting the drivers from "/usr/lib" does not remove them from the already-compressed "initrd.img" file used at boot. You must rebuild it:
 - To update the current kernel's initrd:
      :~$ sudo update-initramfs -u
 - To update all installed kernel images:
      :~$ sudo update-initramfs -u -k all

[3] Verify and Clean Up
 - Remove Xorg remnants: Delete existing X11 configurations that might still try to load the NVIDIA driver:
      :~$ sudo rm /etc/X11/xorg.conf
 - Check for Blacklisting: Ensure the open-source 'Nouveau' driver isn't blacklisted (which would prevent it from loading after NVIDIA is gone). Check for and remove files like "/etc/modprobe.d/blacklist-nouveau.conf".
 - Reboot: Restart your system to boot with the newly generated initrd:
      :~$ sudo reboot

[4] Strategic Recovery
If system is currently in a broken state with partially installed packages, follow these steps:
 - Clean the Slate: Remove all existing NVIDIA packages to prevent dependency conflicts.
      :~$ sudo apt purge ^nvidia-*
 - Use the Recommended Driver Tool: Instead of manually choosing a version, use Ubuntu's Official Ubuntu Drivers tool to identify the most compatible driver for your hardware:
      :~$ sudo ubuntu-drivers devices
      :~$ sudo ubuntu-drivers autoinstall
 - Disable Secure Boot: If the build succeeds but the driver fails to load (check nvidia-smi), Secure Boot might be blocking the unsigned or self-signed kernel module. You may need to disable it in your BIOS/UEFI.
 - Review the Build Log: For specific error details, check the DKMS make log:
      :~$ cat /var/lib/dkms/nvidia/535.x.x/build/make.log

Pro-Tip: If you cannot boot into the desktop to run these commands, hold "Shift + Esc" during boot to enter the 'GRUB' menu, select 'Advanced options', and drop to a 'root shell' in recovery mode to perform the uninstallation.


[Disable Nouveau]
-----------------
To disable the Nouveau driver in Ubuntu, blacklist it by creating /etc/modprobe.d/blacklist-nouveau.conf 
with the lines blacklist nouveau and options nouveau modeset=0. Then, update the initial RAM filesystem 
using sudo update-initramfs -u and reboot your system to apply the changes. 

1) Create a blacklist file: Open a terminal and create a new configuration file to blacklist the driver:
    :~$ echo -e "blacklist nouveau\noptions nouveau modeset=0" | sudo tee /etc/modprobe.d/blacklist-nouveau.conf
  <or>
    :~$ cat <<EOF | sudo tee /etc/modprobe.d/blacklist-nouveau.conf
        blacklist nouveau
        options nouveau modeset=0
        EOF

2) Update initramfs: Update the boot image to ensure the driver does not load during startup:
    :~$ sudo update-initramfs -u

3) Reboot: Restart your computer to apply the changes:
    :~$ sudo reboot

[Nouveau Verification]
After rebooting, you can verify that the Nouveau driver is not loaded by running lsmod. If the command 
returns no output, the driver has been successfully disabled.  
    :~$ lsmod | grep nouveau

[Re-enabling Nouveau]
To reverse this process, delete the configuration file created and update the initramfs again: 
    :~$ sudo rm /etc/modprobe.d/blacklist-nouveau.conf
    :~$ sudo update-initramfs -u
    :~$ sudo reboot
